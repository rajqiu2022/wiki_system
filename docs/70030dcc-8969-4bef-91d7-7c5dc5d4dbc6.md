# ESP32 S3 4G LTE CAT1 A7670X

## 1. Introduction

The Maduino Zero SIM7600 CAT-4 which is based on SIMCOM7600(CAT4) got many 5-stars in the past, but we got much feedback on the price because the CAT-4 module SIM7600 cost high, especially in many IOT project where CAT-4 are not essential. So, now our this new ESP32S3 4G LTE(A7670) have features:
1. Lower Price: this module is based on A7670 4G module, CAT1, its price much lower, while the CAT1 fits for IOT applications such as Remote sensor monitoring;
2. With ESP32S3 controller, it features WIFI/Bluetooth, that can be used for local application setting (with phone) , fit for applications such as shared bike.

Model: [ESPA7670](https://www.makerfabs.com/esp32s3-4g-lte.html)

## 2. Features

- LTE Cat-1, with uplink rate 5 Mbps and downlink rate 10 Mbpsï¼

- GNSS Positioning;
- Controller: ESP32-S3-WROOM-1, PCB Antenna, 16MB Flash, 8MB PSRAM, ESP32-S3-WROOM-1-N16R8;
- 3 USB type-Cï¼ESP32S3 USB Nativeï¼ESP32S3 USB TTLï¼A7670 USB
- Audio Support;
- Supports dial-up, phone, SMS, TCP, UDP, DTMF, HTTP, FTP, and so on;
- USB supply voltage range: 4.8~5.5V, 5.0V Typically;
- Battery supply voltage range: 3.4~4.2V, 3.7V Typically;
- Onboard charger, up to 1A charge current; Overcharge protection(OCP), 4.3V; Over-discharge protection(ODP), 2.5V;
- SD card socket

## 5. Usage

Open the [AT\_demo](https://github.com/Makerfabs/ESP32-S3-4G-LTE-CAT1-A7670X/tree/main/example/at_command_demo) by Arduino IDE.

Verify the code and upload.

Open the serial monitor, select the "Both NL&CR"and send the AT command to the board, and it will return the module's response.

![image.png](https://www.makerfabs.com/image/wiki_image/2023-10-18_18_57_28_0.png)

![image.png](https://www.makerfabs.com/image/wiki_image/2023-10-18_18_58_50_0.png)

```
AT+CPIN?     // Request the state of the SIM card
AT+COPS?     // Check the current network operator
AT+GMR       // Request revision identification
AT+SIMCOMATI // Request a total info of the module
AT+CICCID    // Read ICCID from SIM card
AT+CPSI?     // Inquiring UE system information
AT+CNMP?     // Preferred mode selection, 2-Automatic, 13-GSM Only, 14-WCDMA Only, 38-LTE Only
```

![image.png](https://www.makerfabs.com/image/wiki_image/2023-11-23_16_55_10_0.png)

The more AT commmand details you can cheak [A76XX\_Series\_AT\_Command\_Manual\_V1.10.pdf](https://wikiadmin.makerfabs.com/prod-api/vue-admin-template/user/getFileDownload?file=A76XX_Series_AT_Command_Manual_V1.10_2023-11-22_00_24_40_0.pdf)

### 5.1 Telephone test

```
AT+CNUM          // Request the subscriber number
AT+CSDVC         // Switch voice channel device
AT+CSDVC=1       // 1-Handset, 3-Speaker phone
AT+COUTGAIN=2    // Set loudspeaker volume level to 2, the level range is 0 to 7
ATDxxxxx        // Call to xxxxx
AT+CHUP          // Hang up the call
ATA              // Call answer
```

This is a [demo](https://github.com/Makerfabs/ESP32-S3-4G-LTE-CAT1-A7670X/tree/main/example/ESP32_S3_4G_DIAL_AT_Command) to Telophone test.

Open the sketch by Arduino IDE.

Modify the phone number you want to call.

![image.png](https://www.makerfabs.com/image/wiki_image/2023-11-23_22_10_05_0.png)

Verify the code and upload. When it is successful, the board will automatically dial the target number and hang up within a few seconds.

![image.png](https://www.makerfabs.com/image/wiki_image/2023-11-23_16_57_31_0.png)

You can also dial the number of the SIM card inserted in the board and manually send the "ATA" via the serial monitor, the board will receive your call.

![image.png](https://www.makerfabs.com/image/wiki_image/2023-11-23_16_51_06_0.png)

### 5.2 HTTP test

```
AT+HTTPINIT         // Initialize and start the HTTP
AT+HTTPPARA="URL","http://www.makerfabs.com"  // Set the URL
AT+HTTPACTION=0    // Connect the HTTP. (0-get, 1-post, 2-head)
AT+HTTPHEAD        // Read the response's header.
AT+HTTPREAD=0,500    // Read the content (â500â means the length of data to read)
```

This is a [demo](https://github.com/Makerfabs/ESP32-S3-4G-LTE-CAT1-A7670X/tree/main/example/ESP32_S3_4G_HTTP_AT_Command) to HTTP test.

### 5.3 GPS test

```
AT+CGNSSPWR         //GNSS power control and AP-Flash control
AT+CGNSSPWR=1       //turn the GPS on
AT+CGNSSIPR=9600    //Configure the baud rate of UART3 and GPS module
AT+CGNSSTST=1       //Send data received from UART3 to NMEA port
AT+CGNSSINFO        //Get GNSS fixed position information
```

This is a [demo](https://github.com/Makerfabs/ESP32-S3-4G-LTE-CAT1-A7670X/tree/main/example/ESP32_S3_4G_GPS_AT_Command) to GPS test.

Verify the code and upload. When it is successful, open the serial monitor and the serial port will print the GPS information.
