# LoRaWAN Temperature/ Humidity Soil Moisture Sensor

## 1. Introduction

This module monitor soil moisture via capacitive method, with waterproof coating, it avoids the corrosion problem that usually happens on cheap resistor-mode soil moisture sensor. There is also onboard AHT10 -air temperature/ humidity sensor, to monitor air status, proper for usage in the greenhouse. This module uses 3x AAA battery as power, with sleep current of 26uA and working current of 4mA, the average working time can be more than 1 year. (with 1-hour message interval, Class A ).

This module is mainly for prototyping/ creating, we provide the firmware frames to users, In the firmware frame, we have set the lower power module, PWM setting, etc, users can simply modify a few codes (Appkey/ AppEUI/ DevEUI), to achieve a LoRaWAN application in few minutes. Also, users can modify the setting, such as LoRaWAN message interval, add/ remove functions freely in the codes. Of course, you need to make sure you have a LoRaWAN router in advance.

Comparing the LoRa or LoRawan, please click this [blog link](https://www.makerfabs.cc/article/lorawan-soil-moisture-sensor.html). it maybe helps you to realize them more.

## 2. Features

- LoRaWAN Nodes, RA08H
- Soil Moisture Measurement based on capacitive Testing
- Low power: 7.1uA when sleeping/4mA in message sending turn around
- Onboard AHT10 sensor to monitor air temperature and air humidity
- Onboard Battery Voltage Measurement
- Full Open Source- all hardware and software open at GitHub
- Compatible with Arduino
- Supply power voltage: 4.7~3.5V

When the end device transmits data to TTN, which receives it with the byte stream format. in the **Payload formatters**. Users can configure analytic procedures.

```
function decodeUplink(input) {

  var temp = input.bytes[0]
  var humi = input.bytes[1]
  var adc = input.bytes[2]*256+input.bytes[3]
  var bat = input.bytes[4]/10

  return {
    data:{
    //adc,![LoRaWANTemperatureHumiditySoilMoisturethingspeakchannelconfig.JPG](https://www.makerfabs.com/image/wiki_image/2022-12-24_00_28_35_0.JPG)
    //temp,
    //humi,
    //bat,
    //num,
    field1:adc,
    field2:temp,
    field3:humi,
    field4:bat
    },

  };
  }
```

### 3.3 end device Firmware Setting

Get the Makerfabs LoRaWAN moisture sensor Firmware Frame [Here](https://github.com/Makerfabs/LoraWAN-Soil-Moisture)

Download the file (**Firmware Frame/moisture\_lorawan**) and open it

In this frame, it not only deals the LoRaWAN protocol related but also other functions such as sleep mode setting, PWM generating, air temperature /humidity sensoring... users can apply or modify the setting by simply modifying a few codes in the firmware frame.

![LoRaWANTemperatureHumiditySoilMoistureprocedure.JPG](https://www.makerfabs.com/image/wiki_image/2022-12-24_00_55_12_0.JPG)

In the **config.h**, users must copy their Setting of **DEVEUI,APPEUI,APPKEY** here.

The SLEEP\_CYCLE is a watch timer procedure. It allows the device to wake up periodically and send data. By the current default setting 450, it is about 60 minutes interval, that is, the controller Atmega328P wakes up every 60 minutes to initiate a LoRaWAN data reporting, with about 8s, then it enters sleeping mode again. Users can modify this setting, to balance the Data interval and module working life-time.

then, we can program the sensor via a UART2USB convertor, with Arduino IDE:
