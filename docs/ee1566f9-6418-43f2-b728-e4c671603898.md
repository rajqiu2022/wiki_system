# Maduino Zero 4G LTE

## 1. Introduction

Maduino Zero 4G LTE integrated two types of the 4G LTE CAT4 module SIM7600A-H or SIM7600E-H, SIM7600A-H/SIM7600E-H is a complete multi-band LTE-FDD/LTE-TDD/HSPA+/UMTS/EDGE/GPRS/GSM module solution in LCC type which supports LTE CAT4 up to 150Mbps for downlink and 50Mbps for uplink data transfer, much faster and popular than 2G/3G.

## 2. Features

- Supports dial-up, phone, SMS, TCP, UDP, DTMF, HTTP, FTP, and so on
- Dual USB Type C port
- Control Via AT Commands
- Board USB supply voltage range: 4.8~5.5V, 5.0V Typical
- Board Battery supply voltage range: 3.4~4.2V, 3.7V Typical
- 3GPP E-UTRA Release 11
- Onboard charger, up to 1A charge current
- Overcharge protection(OCP), 4.3V
- Over-discharge protection(ODP), 2.5V
- Power Manager, the board can be supplied by USB or battery.
- IPEX Antenna, GSM/UMTS/LTE main antenna. UMTS/LTE auxiliary antenna. GNSS antenna
- SMS support
- Audio support
- On boarder controller: ATSAMD21G18A
- Audio Codec: NAU8810
- Level Shifter: TXS0108E
- Windows and Raspberry Pi support
- Qualcomm MDM9x07 Chipset

## 4. Usage

**Warning:Do not operate when in power on (donât plug or unplug the Antenna, SIM Cars, SD Card, in case of short-circuiting that may burn the IC down.)**

### 4.1 Hardware connecttion

1.Plug the SIM card into the board.

2.Plug the GPS antenna into the interface.

3.Plug two 4G-GSM antennas into the main antenna interface and auxiliary one.

4.Plug the headphone with the microphone.

5.Plug the SD card into the SD card slot for SIM7600.

When powering the board and the SIM7600 module working, the onboard STA LED(blue) will turn on.

### 4.2 How to use AT commands

For using AT commands by the serial communication, it needs to upload the sketch to this board. The source code is available on [GitHub.](https://github.com/Makerfabs/Maduino-Zero-4G-LTE/tree/main/example/4G_SD_AT_Commands_V1.0)

- Note that there are few AT-Commands may not return the accurate values as the manual says for the different version of the SIM7600

1.Open the sketch by Arduino IDE.

2.Use Type-C USB cable to connect the board and PC, note plug the cable into the USB-MCU interface.

3.Select "Tool --> Board --> Arduino Zero" and the port.

![Maduino_Zero_4G_LTE_7.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_02_36_09_0.jpg)

4.Verify the code and upload.

5.Open the serial monitor, select the "Both NL&CR"and send the AT command to the board, and it will return the module's response.

6.There are some demos that show how to use the AT commands:

- **AT commands test**

```
    AT+CGMI    // Request manufacturer identification
    AT+CGSN    // Request product serial number identification
    AT+CSUB    // Request the module version and chip
    AT+SIMCOMATI //Request a total info of the module
```

![Maduino_Zero_4G_LTE_8.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_02_38_46_0.jpg)

```
    AT+CPIN?     // Request the state of the SIM card
    AT+CICCID    // Read ICCID from SIM card
    AT+CNUM      // Request the subscriber number
    AT+CNMP?     // Preferred mode selection
    AT+COPS?     // Check the current network operator
```

![Maduino_Zero_4G_LTE_9.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_02_39_31_0.jpg)

```
    AT+IPREX?    // Check local baud rate
    AT+CRESET    // Reset the module
```

- **Get the GNSS location**

```
    AT+CGPS=1      // Start GPS session 
    AT+CGPSINFO    // Get GPS fixed position information
    AT+CGPS=0      // Stop GPS session
```

![Maduino_Zero_4G_LTE_10.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_02_42_20_0.jpg)

- **Send and receive SMS**

**Note that is recommended to use other serial monitors not the Arduino IDE one to send the AT command for this demo.**

```
    AT+CSCA="XXXXXX"       // Set the SMS service centre address(it isn't your SIM number)
    AT+CMGF=1              // Select SMS message format
    AT+CMGS="xxxxxx"       // Send message to "xxxxxx"(the receiver number).
```

After sending the above AT commands, it will show ">" and then you can send your message. When you finish your message, you need to send "1A" with hexadecimal for confirmation or send "1B" with hexadecimal for cancellation, that is why recommended to use other serial monitors.

![Maduino_Zero_4G_LTE_11.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_02_59_37_0.jpg)

```
    AT+CMGR=3        // Read message
    AT+CMGD=3        // Delete message
```

![Maduino_Zero_4G_LTE_12.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_03_02_37_0.jpg)

- **Make a call**

```
    AT+CSDVC         // Switch voice channel device
    AT+CSDVC=1       // 1-Handset, 3-Speaker phone
    AT+CLVL=2        // Set loudspeaker volume level to 2, the level range is 0 to 5
```

![Maduino_Zero_4G_LTE_13.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_03_09_14_0.jpg)

```
    ATDxxxxx;        // Call to xxxxx
    AT+CHUP          // Hang up the call
```

![Maduino_Zero_4G_LTE_14.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_03_10_03_0.jpg)

```
    AT+CLIP=1     // Calling line identification presentation
    ATA         // Call answer
```

![Maduino_Zero_4G_LTE_15.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_03_11_13_0.jpg)

- **HTTP test**

```
    AT+HTTPINIT         // Initialize and start the HTTP
    AT+HTTPPARA="URL","http://www.makerfabs.com"  // Set the URL
    AT+HTTPACTION=0    // Connect the HTTP. (0-get, 1-post, 2-head)
    AT+HTTPHEAD        // Read the response's header.
    AT+HTTPREAD=0,3    // Read the content (â3â means the number of the reading data)
```

![Maduino_Zero_4G_LTE_16.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_18_22_15_0.jpg)

- **Test the SD card for SIM7600**

When you plug an SD card into the SD card slot for SIM7600, You can use the following commands to check it.

```
    AT+FSCD=D:                    // Select SD card directory as current directory
    AT+FSLS                       // List directories/files in current directory
    AT+CFTRANRX="D:/TEST.txt",9  // Transfer a file to EFS, input 9 letters after returns ">" ,such as Makerfabs
    AT+CFTRANTX="D:/TEST.txt"     // Transfer a file from EFS to host
```

- **More AT commands detail please check [SIM7500\_SIM7600\_Series\_AT\_Command\_Manual\_V1.10.pdf](https://wikiadmin.makerfabs.com/prod-api/vue-admin-template/user/getFileDownload?file=SIM7500_SIM7600_Series_AT_Command_Manual_V1.10_2023-11-25_01_20_07_0.pdf)**

### 4.4 Surfing the internet

Maduino Zero 4G LTE can be a wireless networking device to support the PC or Raspberry PI surfing the internet. Next, it will show how to do it.

- It needs to install the SIM7600 driver to the PC, the driver is available on [resources](https://github.com/Makerfabs/Maduino-Zero-4G-LTE/tree/main/win_driver)
- Use Type-C USB cable to connect the board(USB-LTE) and PC, note plug the cable into the USB of MCU.

![Maduino_Zero_4G_LTE_18.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_18_48_10_0.jpg)

![Maduino_Zero_4G_LTE_19.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_18_48_18_0.jpg)

- Download the driver to the PC.
- Follow the below step to install the driver manually:

1.Open "Device Manager".

2.In "Device Manager", select "Other devices".

3.Right-click on and click "Update Driver Software".

![Maduino_Zero_4G_LTE_20.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_18_50_43_0.jpg)

4.Select "Browse my computer for driver software".

![Maduino_Zero_4G_LTE_21.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_18_51_25_0.jpg)

5.Paste the location address of the driver into the location field, then click "Next".

6.Wait until the drivers' installation process completes.

![Maduino_Zero_4G_LTE_22.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_18_52_11_0.jpg)

Note: Install all the drivers that shows as yellow exclamation mark.
- If the PC does not connect to the internet through this device, please open the serial monitor and send the AT command to start the networking.

```
    AT$QCRMCALL=1,1
```

If have not yet, please use the PPP dial-up connection way to start networking.

More info you can get from [GitHub](https://github.com/Makerfabs/Maduino-Zero-4G-LTE#usage-windows)

### 4.5 Transmit T&H to Thingspeak

This demo will show you use this device to transmit the temperature and humidity to Thingspeak that the T&H CAN be checked everywhere.

- Prepare a [DHT11 module](https://www.makerfabs.com/mabee-dht11.html) for detecting the temperature and humidity.
- Follow the wiring to connect the DHT11 module and the board.

| DHT11 | Maduino Zero 4G LTE |
| --- | --- |
| VCC | 3V3 |
| GND | GND |
| DATA | D3 |

![Maduino_Zero_4G_LTE_23.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_18_57_48_0.jpg)

- The sketch for this demo is available on [Github](https://github.com/Makerfabs/Maduino-Zero-4G-LTE/tree/main/example/4G_HTTP_DHT11_V1.0).
- To transmit the data to your [Thingspeak](https://thingspeak.com/channels/1156784) channel, it has to create a new channel and get the APIKEY information. Then replace the APIKEY in the code with yours.

![Maduino_Zero_4G_LTE_24.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_19_01_09_0.jpg)

- Open the sketch by Arduino IDE.
- After connecting the antenna and SIM card, plug the cable into the USB-MCU interface and connect - with the PC.
- Select "Tool --> Board --> Arduino Zero" and the port.
- Install the library for DHT11 module.

![Maduino_Zero_4G_LTE_25.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_19_10_07_0.jpg)

- Upload the sketch to the board.
- The device will transmit the data to the internet every 5 seconds, you can check the temperature and humidity on the Thingspeak channel.

![Maduino_Zero_4G_LTE_26.jpg](https://www.makerfabs.com/image/wiki_image/2022-10-24_19_11_09_0.jpg)

Q:When using 3.3V pull-up resistors, the input values of D8 are sometimes low and sometimes high, are there some problems with it ?
A:when you turn on the SIM7600 module ,D8 and D9 from the SAMD are connected to RI and DTR on the SIM7600 through level shifter TXS1080E, as part of the UART interface. when receiving SMS, or a URC report is coming, the DTR pin can be used to wake module from sleep and RI ,for getting the low level .
